Azure App Registration Guide — Mail.Send via Client Credentials
  1. 1
    Go to portal.azure.com → Microsoft Entra ID → App registrations → New registration. Name it something descriptive like ProxMigrate Mail. Supported account types: Accounts in this organizational directory only. No redirect URI is needed for this app — it uses application permissions, not user sign-in.
  2. 2
    After creation, copy the Application (client) ID and Directory (tenant) ID from the Overview page — you will need both below.
  3. 3
    Go to API permissions → Add a permission → Microsoft Graph → Application permissions (not Delegated — this is server-to-server) and add the following:
    Permission Type Admin consent Purpose
    Mail.Send Application Required Allows ProxMigrate to send email as the configured From mailbox

    After adding Mail.Send, click Grant admin consent for <your tenant>. The status column must show a green Granted tick — the permission will not work otherwise.

  4. 4
    Restrict the permission to a specific mailbox (recommended). By default, Mail.Send as an Application permission allows the app to send as any mailbox in your tenant. To restrict it to a single mailbox, create an Application Access Policy in Exchange Online PowerShell: 
    # Run in Exchange Online PowerShell
New-ApplicationAccessPolicy `
  -AppId "<your-client-id>" `
  -PolicyScopeGroupId "<mailbox@yourcompany.com>" `
  -AccessRight RestrictAccess `
  -Description "Restrict ProxMigrate to send from one mailbox"

    This is optional but strongly recommended for production. A shared mailbox (e.g. alerts@yourcompany.com) works well as the From address.

  5. 5
    Go to Certificates & secrets → Client secrets → New client secret. Set an expiry, then copy the secret Value immediately — Azure only shows it once. Paste it into the Client Secret field below.
  6. 6
    Fill in the three fields below (Tenant ID, Client ID, Client Secret), set the From Address to the mailbox you want to send from, save, and use Send Test to verify everything works end-to-end.
