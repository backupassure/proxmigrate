Export Modes Explained
Shutdown

Cleanest option. The VM is gracefully stopped, the disk is copied to a qcow2, then the VM is optionally restarted. No dirty data, no journal replay — identical to a clean shutdown and backup. Best for production databases (MySQL, PostgreSQL, SQL Server) and Windows.

Filesystem Freeze

Uses the QEMU guest agent to pause all disk I/O before copying starts, then resumes immediately after. The VM stays running — users won't notice a reboot — but applications will stall briefly during the copy. Duration depends on disk size.

Requires QEMU guest agent installed and running in the VM.

Live (crash-consistent)

Fastest — no downtime at all. Reads the disk while the VM runs, equivalent to an unexpected power cut at that moment. Linux journaled filesystems recover automatically on next boot. Not recommended for Windows or active databases.
