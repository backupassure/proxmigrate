{# ===== Tab 1: Local Accounts ===== #}
Local Authentication
Local authentication is always enabled and cannot be disabled. It serves as the fallback for administrator access.

Local accounts are stored in ProxMigrate's database. Use the User Management page to create, edit, and deactivate local users.

Manage Local Users
{# ===== Tab: MFA ===== #}
Multi-Factor Authentication

MFA adds a second layer of security by requiring an authenticator app code (Google Authenticator, Authy, Microsoft Authenticator) after username and password. Applies to Local and LDAP accounts. Entra ID users are excluded — MFA is managed by their identity provider.

Global Enforcement

{% if mfa_enforced %} All local and LDAP users are required to set up MFA on their next login. Users cannot disable MFA while enforcement is active. {% else %} MFA is optional. Users can enable it from the MFA page in their account settings. {% endif %}

{# Azure App Registration guide #}
Azure App Registration Guide
  1. 1
    Go to portal.azure.com → Microsoft Entra ID → App registrations → New registration.
  2. 2
    Name: ProxMigrate. Supported account types: Accounts in this organizational directory only (or multi-tenant if needed). Add the Redirect URI shown above.
  3. 3
    After creation, copy the Application (client) ID and Directory (tenant) ID from the overview page.
  4. 4
    Go to API permissions → Add a permission → Microsoft Graph → Delegated permissions and confirm the following are present:
    Permission Type Admin consent Purpose
    User.Read Delegated Not required Sign in, read user profile & email — added by default
    GroupMember.Read.All Delegated Required Check group membership — required if Require Group ID or Admin Group ID is set

    After adding permissions, click Grant admin consent for <your tenant> to activate the GroupMember.Read.All permission.

  5. 5
    Go to Certificates & secrets → New client secret. Copy the secret Value (shown once only).
